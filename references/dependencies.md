# Проверка 2 — Уязвимые зависимости

> Твой код может быть идеален, но если библиотека внутри него дырявая — дырявый и
> ты. Большинство взломов идут через известные (уже опубликованные) уязвимости в
> зависимостях.

## Автосканер

```bash
node scripts/deps-audit.mjs .
```
Сам определяет экосистему и запускает нужный аудитор. Ничего не устанавливает и не
меняет — только читает.

## По экосистемам (если автосканер не покрыл)

| Файл-маркер | Команда аудита |
|---|---|
| `package.json` + `package-lock.json` | `npm audit` |
| `pnpm-lock.yaml` | `pnpm audit` |
| `yarn.lock` | `yarn npm audit` |
| `requirements.txt` / `pyproject.toml` | `pip-audit` (или `safety check`) |
| `go.mod` | `govulncheck ./...` |
| `Cargo.lock` | `cargo audit` |
| `composer.lock` | `composer audit` |
| `Gemfile.lock` | `bundle audit` |

## Как читать вывод

- **critical / high** → 🔴. Чинить в первую очередь, особенно если пакет в проде.
- **moderate** → 🟡. Запланировать обновление.
- **low / info** → 🟢. Учесть, но без паники.
- **Нет lockfile** → аудит неполный. Зафиксируй lockfile (`npm install` создаёт
  `package-lock.json`) и закоммить его — иначе у всех разные версии и аудит врёт.

## Как починить (шаги для пользователя)

1. **Автофикс безопасных обновлений:** `npm audit fix` (или аналог). Обновляет в
   рамках совместимых версий.
2. **Если остались critical/high**, требующие мажорного обновления:
   `npm audit fix --force` — но потом ОБЯЗАТЕЛЬНО прогони сборку и тесты
   (`npm run build`), мажор может сломать API.
3. **Если фикса нет** (мейнтейнер не выпустил патч) — оцени, используешь ли ты
   уязвимую функцию; при необходимости замени пакет на живой аналог.
4. На будущее: включи Dependabot / Renovate в репозитории — он сам будет открывать
   PR с обновлениями безопасности.

## Частые сопутствующие проблемы

- [ ] **Лишние зависимости.** Чем больше пакетов — тем больше поверхность атаки.
      Убери неиспользуемые.
- [ ] **Заброшенные пакеты** (нет коммитов годами) — кандидаты на замену.
- [ ] **Подозрительные пакеты** (опечатка в имени популярного — typosquatting).
      Проверь, что имена в `package.json` — те, что ты осознанно ставил.
