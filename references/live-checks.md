# Шаг 3 — Live-проверки опубликованного проекта (опционально)

> Только для **своего** ресурса (или с явного разрешения владельца). Только
> неразрушающие GET-проверки. Никакого перебора, нагрузки, эксплойтов, сканирования
> чужих хостов. Если пользователь не дал ссылку — пропусти этот шаг.

Сначала спроси: «Проект уже опубликован? Дай ссылку (URL сайта, @username бота или
адрес API) — проверю живую версию.»

---

## Сайт / веб-приложение

Проверяй вежливо (несколько запросов, не нагрузка):

- [ ] **HTTPS и редирект.** `http://` редиректит на `https://`? Сертификат валиден?
- [ ] **Заголовки безопасности.** Глянь ответ: есть ли `Strict-Transport-Security`,
      `Content-Security-Policy`, `X-Content-Type-Options`, `X-Frame-Options`/
      `frame-ancestors`, `Referrer-Policy`. Чего нет — в отчёт (🟡).
- [ ] **Не отдаётся ли лишнее:**
      `/.env`, `/.git/config`, `/.git/HEAD`, `/config.json`, `/backup.zip`,
      `/.DS_Store`, `/phpinfo.php` — должны давать 404, не 200. 🔴 если отдаются.
- [ ] **Листинг директорий** выключен (`/uploads/`, `/assets/` не показывают список).
- [ ] **Server-баннеры** не светят лишних версий (`Server: nginx/1.18.0` — мелочь,
      но в отчёт как 🟢).
- [ ] **Страницы ошибок** не показывают стектрейс (открой заведомо битый URL).
- [ ] **Админка/служебные пути** (`/admin`, `/wp-admin`, `/.well-known`) — закрыты/
      под авторизацией.

Способ проверки (пример, безопасно):
```bash
# заголовки
curl -sI https://example.com | sed -n '1,20p'
# не отдаётся ли секретное (ждём 404)
for p in /.env /.git/HEAD /config.json /backup.zip; do
  echo -n "$p → "; curl -s -o /dev/null -w "%{http_code}\n" "https://example.com$p"
done
```
> Если в окружении агента `curl` ограничен — используй встроенный HTTP-инструмент
> (fetch/WebFetch) тем же набором путей. Логика та же: 200 на секретном пути = 🔴.

---

## API

- [ ] **Защищённые эндпоинты без токена** дают 401/403, а не данные.
- [ ] **CORS-заголовки** в ответе не `*` для приватного API с credentials.
- [ ] **Ошибки** — общие, без стектрейса/SQL.
- [ ] **`OPTIONS`/несуществующие методы** обрабатываются корректно.
- (Не делай нагрузочных и переборных проверок.)

---

## Telegram-бот

- [ ] **Реакция на чужие команды.** Напиши боту как обычный пользователь —
      админ-команды (`/broadcast`, `/stats`, `/admin`) должны отвечать «нет доступа»,
      а не выполняться.
- [ ] **Нет утечки служебной инфы** в ответах (пути, токены, ошибки сервера).
- [ ] **Webhook-эндпоинт** (если известен) не принимает апдейты без `secret_token`.
- (Проверяй только своего бота, по-человечески, без флуда.)

---

## Что НЕ делаем

- ❌ Брутфорс паролей/токенов.
- ❌ Нагрузочное/DoS-тестирование.
- ❌ Реальную эксплуатацию найденных дыр.
- ❌ Любые проверки чужих ресурсов без разрешения.

Нашёл признак проблемы → зафиксируй как находку с пруфом (путь + код ответа) и
опиши в отчёте, как закрыть. Доказывать эксплойтом не нужно.
