# Проверка 4 — Конфиги и деплой

> Часто код в порядке, а дыра — в настройках: открытый порт, `DEBUG=true` в проде,
> CORS «всем можно», секрет в CI-логах.

## Чеклист — репозиторий и окружение

- [ ] **`.env` / секреты не в репо** (см. также `secrets.md`).
- [ ] **`.env.example` есть** и содержит ПУСТЫЕ значения — чтобы знать, что нужно,
      не раскрывая секретов.
- [ ] **`DEBUG` / verbose-режим выключены в проде.** Django `DEBUG=False`,
      Flask без `debug=True`, Node `NODE_ENV=production`. В дебаге наружу летят
      стектрейсы с путями и переменными.
- [ ] **Ошибки не показывают стектрейс/SQL пользователю.** Наружу — общее сообщение,
      детали — только в лог.

## Чеклист — CORS и заголовки

- [ ] **CORS не `*` для приватного API.** `Access-Control-Allow-Origin: *` вместе с
      `credentials` — дыра. Перечисли разрешённые домены.
- [ ] **Заголовки безопасности** (для сайтов/API):
      `Strict-Transport-Security` (HSTS), `Content-Security-Policy` (CSP),
      `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY` (или CSP
      `frame-ancestors`), `Referrer-Policy`.
- [ ] **Cookie с флагами** `Secure; HttpOnly; SameSite` (см. `auth-access.md`).

## Чеклист — сеть и сервер (если есть доступ/конфиги)

- [ ] **Наружу торчат только нужные порты.** БД (5432/3306/27017), Redis (6379),
      админки — НЕ должны слушать `0.0.0.0` без файрвола. 🔴 если БД открыта в интернет.
- [ ] **HTTPS включён**, HTTP редиректит на HTTPS, сертификат валиден.
- [ ] **Дефолтные пароли служб сменены** (БД, Redis, админ-панели, Grafana и т.п.).
- [ ] **Бэкапы** настроены и сами не лежат в публичном доступе.

## Чеклист — CI/CD (`.github/workflows`, GitLab CI и т.п.)

- [ ] **Секреты в CI — через Secrets/Variables**, не в `yml` открытым текстом.
- [ ] **Секреты не печатаются в логи** (`echo $TOKEN` в шаге = утечка в публичный лог).
- [ ] **`pull_request` из форков не получает доступ к секретам** (для публичных репо).
- [ ] **Действия (actions) запинены** на версию/SHA, а не плавающий `@master`.
- [ ] **Деплой-ключи** имеют минимальные права (deploy key на 1 репо, не аккаунт).

## Чеклист — Docker (если есть `Dockerfile`)

- [ ] **Контейнер не от root** без необходимости (`USER` в Dockerfile).
- [ ] **В образ не копируются `.env`, ключи, `.git`** (проверь `.dockerignore`).
- [ ] **Базовый образ свежий** и из доверенного источника (не случайный `latest`).
- [ ] **Секреты не в `ENV`/слоях образа** (видны через `docker history`).

## Severity-ориентир

- БД/Redis открыты в интернет без пароля → 🔴.
- `DEBUG=true` в проде, секрет в CI-логах → 🔴.
- CORS `*` на приватном API с credentials → 🔴.
- Нет HSTS/CSP → 🟡.
- Контейнер от root → 🟡.

## Как починить

1. Вынеси все секреты в `.env` (локально) и в Secrets CI (на проде). Добавь
   `.dockerignore` с `.env`, `.git`, `node_modules`.
2. Выключи debug, скрой стектрейсы наружу.
3. Закрой лишние порты файрволом (ufw / security groups), оставь 80/443.
4. Настрой заголовки безопасности (в nginx, Cloudflare или фреймворке).
5. Переключи CORS на белый список доменов.
